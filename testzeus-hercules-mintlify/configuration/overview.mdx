## Quick Configuration

### Essential Settings

Get started quickly with these essential configuration settings:

```bash
# LLM Provider (Required)
export OPENAI_API_KEY="your-api-key"

# Browser Settings
export BROWSER_TYPE="chromium"
export HEADLESS="false"

# Timeout Settings
export TIMEOUT="30000"
export NAVIGATION_TIMEOUT="10000"
```

### Configuration File

Create a `hercules.config.json` file for persistent settings:

```json
{
  "llm": {
    "provider": "openai",
    "model": "gpt-4",
    "temperature": 0.1
  },
  "browser": {
    "type": "chromium",
    "headless": false,
    "viewport": {
      "width": 1920,
      "height": 1080
    }
  },
  "timeouts": {
    "default": 30000,
    "navigation": 10000
  },
  "debug": {
    "enabled": true,
    "screenshot_on_failure": true,
    "save_logs": true
  }
}
```

## Configuration Hierarchy

TestZeus Hercules follows a configuration hierarchy where settings are applied in this order (later sources override earlier ones):

1. **Default values** - Built-in defaults
2. **Configuration file** - `hercules.config.json`
3. **Environment variables** - System environment variables
4. **Command-line arguments** - Runtime arguments

### Example Configuration Hierarchy

```bash
# 1. Default: headless=true
# 2. Config file: headless=false (overrides default)
# 3. Environment: HEADLESS=true (overrides config file)
# 4. Command line: --headless=false (overrides environment)

hercules --headless=false test.feature
```

## Browser Configuration

### Browser Types

TestZeus Hercules supports multiple browsers through Playwright:

```bash
# Chromium (recommended)
export BROWSER_TYPE="chromium"

# Firefox
export BROWSER_TYPE="firefox"

# WebKit (Safari engine)
export BROWSER_TYPE="webkit"
```

### Browser Options

```bash
# Headless mode (true/false)
export HEADLESS="false"

# Viewport size
export VIEWPORT_WIDTH="1920"
export VIEWPORT_HEIGHT="1080"

# Device emulation
export DEVICE_NAME="iPhone 12"

# User agent
export USER_AGENT="custom-user-agent"
```

### Advanced Browser Settings

```json
{
  "browser": {
    "type": "chromium",
    "headless": false,
    "viewport": {
      "width": 1920,
      "height": 1080
    },
    "device": "Desktop",
    "userAgent": "Mozilla/5.0...",
    "locale": "en-US",
    "timezone": "America/New_York",
    "permissions": ["geolocation", "notifications"],
    "args": [
      "--disable-web-security",
      "--disable-features=VizDisplayCompositor"
    ]
  }
}
```

## Timeout Configuration

### Default Timeouts

```bash
# General timeout (30 seconds)
export TIMEOUT="30000"

# Navigation timeout (10 seconds)
export NAVIGATION_TIMEOUT="10000"

# Element wait timeout (5 seconds)
export ELEMENT_TIMEOUT="5000"

# API request timeout (15 seconds)
export API_TIMEOUT="15000"
```

### Timeout Configuration File

```json
{
  "timeouts": {
    "default": 30000,
    "navigation": 10000,
    "element": 5000,
    "api": 15000,
    "screenshot": 5000,
    "download": 30000
  }
}
```

## Debug and Logging Configuration

### Debug Settings

```bash
# Enable debug mode
export DEBUG="true"

# Screenshot on failure
export SCREENSHOT_ON_FAILURE="true"

# Save execution logs
export SAVE_LOGS="true"

# Log level (debug, info, warn, error)
export LOG_LEVEL="info"

# Log output directory
export LOG_DIR="./logs"
```

### Logging Configuration

```json
{
  "debug": {
    "enabled": true,
    "screenshot_on_failure": true,
    "save_logs": true,
    "log_level": "info",
    "log_dir": "./logs",
    "verbose": false
  }
}
```

## Test Execution Configuration

### Execution Modes

```bash
# Single test mode (default)
export EXECUTION_MODE="single"

# Bulk test mode
export EXECUTION_MODE="bulk"

# Parallel execution
export PARALLEL_WORKERS="4"

# Test retry count
export RETRY_COUNT="3"
```

### Test Filtering

```bash
# Run tests with specific tags
export TEST_TAGS="@smoke,@critical"

# Exclude tests with tags
export EXCLUDE_TAGS="@slow,@manual"

# Test environment
export TEST_ENVIRONMENT="staging"
```

## Configuration Validation

### Validate Configuration

TestZeus Hercules provides commands to validate your configuration:

```bash
# Check configuration
hercules config check

# Show current configuration
hercules config show

# Test LLM connection
hercules config test-llm

# Test browser setup
hercules config test-browser
```

### Configuration Errors

Common configuration issues and solutions:

<AccordionGroup>
  <Accordion title="Invalid LLM API key">
    ```bash
    # Check API key format
    echo $OPENAI_API_KEY | wc -c  # Should be ~51 characters
    
    # Test API key
    hercules config test-llm
    ```
  </Accordion>

  <Accordion title="Browser not found">
    ```bash
    # Install browsers
    playwright install
    
    # Check browser installation
    playwright --version
    ```
  </Accordion>

  <Accordion title="Invalid timeout values">
    ```bash
    # Timeouts should be in milliseconds
    export TIMEOUT="30000"  # 30 seconds
    export NAVIGATION_TIMEOUT="10000"  # 10 seconds
    ```
  </Accordion>

  <Accordion title="Configuration file not found">
    ```bash
    # Create default configuration
    hercules config init
    
    # Specify custom config file
    hercules --config custom-config.json test.feature
    ```
  </Accordion>
</AccordionGroup>

## Best Practices

### Configuration Management

1. **Use Environment Variables**: For sensitive data like API keys
2. **Configuration Files**: For complex settings and team sharing
3. **Environment-Specific Configs**: Different settings for dev/staging/prod
4. **Version Control**: Include config templates, exclude sensitive data
5. **Documentation**: Document all configuration options

### Security Best Practices

1. **Never commit API keys** to version control
2. **Use environment variables** for sensitive configuration
3. **Encrypt sensitive data** when possible
4. **Rotate API keys** regularly
5. **Use least privilege** access for API keys

### Performance Optimization

1. **Adjust timeouts** based on your application's performance
2. **Use parallel execution** for faster test runs
3. **Enable caching** for repeated operations
4. **Optimize browser settings** for your use case
5. **Monitor resource usage** and adjust limits accordingly
