# Environment Variables API

This page provides a comprehensive reference for all environment variables supported by TestZeus Hercules, organized by category and functionality.

## LLM Configuration

### OpenAI Configuration

```bash
# OpenAI API settings
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-4o  # Default model
OPENAI_BASE_URL=https://api.openai.com/v1  # Custom endpoint
OPENAI_TIMEOUT=60  # Request timeout in seconds
OPENAI_MAX_RETRIES=3  # Maximum retry attempts
```

## Configuration Validation

### Required Variables

The following environment variables are required for basic operation:

- At least one LLM provider API key (OPENAI_API_KEY, ANTHROPIC_API_KEY, etc.)
- BROWSER (defaults to 'chrome' if not specified)

### Optional Variables

All other environment variables are optional and have sensible defaults.

### Validation Rules

```bash
# Environment validation
VALIDATE_CONFIG=true  # Enable configuration validation
STRICT_VALIDATION=false  # Enable strict validation mode
CONFIG_SCHEMA_PATH=./config/schema.json  # Configuration schema
```

## Best Practices

### Security Best Practices

1. **Never commit API keys** to version control
2. **Use environment files** for local development
3. **Rotate API keys** regularly in production
4. **Encrypt sensitive data** at rest

### Performance Best Practices

1. **Set appropriate timeouts** for your use case
2. **Enable caching** for repeated operations
3. **Monitor memory usage** in long-running tests
4. **Use headless mode** for faster execution

### Configuration Management

1. **Use configuration files** for complex setups
2. **Document custom variables** in your project
3. **Validate configuration** before test execution
4. **Use environment-specific configs** for different stages
