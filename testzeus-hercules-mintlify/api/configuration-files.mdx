## Configuration File Discovery

TestZeus Hercules automatically discovers configuration files in this order:

1. `hercules.config.js`
2. `hercules.config.json`
3. `hercules.config.yaml` or `hercules.config.yml`
4. `.herculesrc.json`
5. `.herculesrc.yaml` or `.herculesrc.yml`
6. `package.json` (in `hercules` field)

You can also specify a custom configuration file:

```bash
hercules --config custom-config.json test.feature
```

### Dynamic JavaScript Configuration

```javascript
// hercules.config.js
const path = require('path');
const os = require('os');

const isCI = process.env.CI === 'true';
const isDevelopment = process.env.NODE_ENV === 'development';

module.exports = {
  // Dynamic LLM configuration based on environment
  llm: {
    provider: process.env.LLM_PROVIDER || 'openai',
    model: isCI ? 'gpt-3.5-turbo' : 'gpt-4',
    temperature: 0.1,
    timeout: isCI ? 60000 : 30000
  },
  
  // Dynamic browser configuration
  browser: {
    type: 'chromium',
    headless: isCI || !isDevelopment,
    viewport: {
      width: 1920,
      height: 1080
    },
    args: isCI ? [
      '--no-sandbox',
      '--disable-setuid-sandbox',
      '--disable-dev-shm-usage'
    ] : []
  },
  
  // Environment-specific timeouts
  timeouts: {
    default: isCI ? 60000 : 30000,
    navigation: isCI ? 30000 : 10000,
    element: 5000
  },
  
  // Dynamic execution configuration
  execution: {
    mode: isCI ? 'bulk' : 'single',
    parallelWorkers: isCI ? Math.min(os.cpus().length, 4) : 1,
    retryCount: isCI ? 2 : 0,
    failFast: isCI
  },
  
  // Environment-specific debug settings
  debug: {
    enabled: isDevelopment,
    screenshotOnFailure: true,
    saveLogs: true,
    logLevel: isDevelopment ? 'debug' : 'info',
    logDir: path.join(__dirname, 'logs')
  },
  
  // Dynamic reporting configuration
  reporting: {
    enabled: true,
    formats: isCI ? ['junit', 'json'] : ['html'],
    outputDir: path.join(__dirname, 'reports'),
    filename: `test-report-${new Date().toISOString().split('T')[0]}`,
    includeScreenshots: true,
    includeLogs: isDevelopment
  },
  
  // Conditional feature enablement
  accessibility: {
    enabled: process.env.ENABLE_ACCESSIBILITY_TESTS === 'true',
    standard: 'WCAG21AA'
  },
  
  visual: {
    enabled: process.env.ENABLE_VISUAL_TESTS === 'true',
    threshold: parseFloat(process.env.VISUAL_THRESHOLD) || 0.1
  },
  
  // Environment-specific API configuration
  api: {
    baseUrl: process.env.API_BASE_URL || 'https://api.example.com',
    timeout: 15000,
    retryCount: isCI ? 3 : 1
  },
  
  // Conditional notifications
  notifications: {
    enabled: isCI,
    onFailure: true,
    slack: {
      webhookUrl: process.env.SLACK_WEBHOOK_URL,
      channel: process.env.SLACK_CHANNEL || '#testing'
    }
  }
};
```

### Advanced JavaScript Configuration with Functions

```javascript
// hercules.config.js
const { execSync } = require('child_process');

function getGitBranch() {
  try {
    return execSync('git rev-parse --abbrev-ref HEAD', { encoding: 'utf8' }).trim();
  } catch {
    return 'unknown';
  }
}

function getTestEnvironment() {
  const branch = getGitBranch();
  if (branch === 'main' || branch === 'master') return 'production';
  if (branch.startsWith('release/')) return 'staging';
  return 'development';
}

module.exports = {
  // Environment detection
  environment: getTestEnvironment(),
  
  // Branch-specific configuration
  llm: {
    provider: 'openai',
    model: getTestEnvironment() === 'production' ? 'gpt-4' : 'gpt-3.5-turbo'
  },
  
  // Dynamic test selection
  execution: {
    tags: (() => {
      const env = getTestEnvironment();
      switch (env) {
        case 'production': return '@smoke';
        case 'staging': return '@smoke,@regression';
        default: return '';
      }
    })()
  },
  
  // Custom hooks
  hooks: {
    beforeAll: async () => {
      console.log(`Running tests in ${getTestEnvironment()} environment`);
    },
    
    afterAll: async (results) => {
      if (results.failed > 0 && getTestEnvironment() === 'production') {
        // Send alert for production failures
        console.log('Production test failures detected!');
      }
    }
  }
};
```

## Configuration Validation

### Built-in Validation

TestZeus Hercules automatically validates configuration files:

```bash
# Validate current configuration
hercules config validate

# Validate specific configuration file
hercules config validate --config custom-config.json

# Show validation errors
hercules config validate --verbose
```

### Custom Validation

```javascript
// hercules.config.js with validation
const config = {
  llm: {
    provider: 'openai',
    model: 'gpt-4'
  },
  browser: {
    type: 'chromium',
    headless: false
  }
};

// Custom validation function
function validateConfig(config) {
  const errors = [];
  
  if (!config.llm?.provider) {
    errors.push('LLM provider is required');
  }
  
  if (config.browser?.viewport?.width < 320) {
    errors.push('Viewport width must be at least 320px');
  }
  
  if (errors.length > 0) {
    throw new Error(`Configuration validation failed:\n${errors.join('\n')}`);
  }
  
  return config;
}

module.exports = validateConfig(config);
```

## Best Practices

### Configuration Management

1. **Use environment-specific configurations** for different deployment stages
2. **Keep sensitive data in environment variables**, not configuration files
3. **Use configuration inheritance** to avoid duplication
4. **Validate configurations** before running tests
5. **Document all configuration options** used in your project

### Security Best Practices

1. **Never commit API keys** to version control
2. **Use environment variables** for sensitive configuration
3. **Encrypt sensitive configuration** when possible
4. **Use secure storage** for production configurations
5. **Rotate credentials** regularly

### Performance Optimization

1. **Adjust timeouts** based on your application's performance
2. **Use appropriate parallel worker counts** for your environment
3. **Enable caching** for repeated operations
4. **Optimize browser settings** for your use case
5. **Monitor resource usage** and adjust limits accordingly

### Maintenance

1. **Keep configurations up to date** with framework changes
2. **Review and update** configurations regularly
3. **Test configurations** in different environments
4. **Document configuration changes** and their impact
5. **Use version control** for configuration files
